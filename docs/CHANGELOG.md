## [2.1.0] — 2026-03-15

### Новое
- Фильтр по глаголам (5-й уровень фильтрации «Глагол»): сворачиваемая секция с многоколоночной сеткой (6 колонок десктоп, 4 планшет, 3 мобильные), алфавитный порядок сверху вниз, badge-счётчик с кнопкой сброса (✕).
- Навигационные стрелки (← →) в блоке спряжения рядом с кнопками «Проверить» / «Повторить».
- Два новых API-эндпоинта: `GET /api/all-verbs`, `GET /api/verb-category-map`.

### Улучшено
- Контекст глагола разделён на 2 строки (наклонение + время, категория + тип) для предотвращения скачков раскладки.
- Правильные ответы отображаются над полем ввода (в строке метки лица), зелёным жирным шрифтом, без префикса «Правильно:».
- Блок «Результат: X из Y» всегда присутствует в DOM (`visibility: hidden` при отсутствии результатов) — без скачков раскладки.
- Badge-счётчик фильтра глаголов всегда в DOM (`opacity: 0` при пустом выборе) — без скачков раскладки.
- Placeholder «Введите спряжение…» стал светлее (`#ccc`).
- Обработка NULL-форм: пустой ввод, пробел или прочерк = правильный ответ; любой другой ввод = ошибка с подсказкой «–».

### Мобильные оптимизации
- Навигационные кнопки: уменьшенный шрифт (11px) и padding на мобильных.
- Инфинитив глагола: 26px на мобильных (вместо 36px).
- Поля ввода: меньше padding и шрифт (14px) на мобильных.
- Карточки: меньше padding и gap на мобильных.
- Навигационные стрелки: 32px на мобильных (вместо 36px).

## [2.0.0] — 2026-03-14

### Новое
- Полностью переработанный frontend на React 19 + Vite 8 + TypeScript.
- Карточная вёрстка с градиентным фоном страницы.
- Кросс-фильтрация: недоступные значения фильтров отображаются как disabled.
- Кнопки фильтров с градиентной подсветкой активного состояния.
- Блок спряжения: одноколоночная раскладка, центрирование.
- Подсказки правильных форм при неверных ответах.
- Корректная обработка NULL-форм: пустой ввод/прочерк = правильно, иначе «Правильно: –».
- Результат проверки отображается под кнопками.
- Кнопки «Проверить» и «Повторить» всегда доступны.
- Vite proxy для запросов `/api` к backend.
- Пользовательский порядок фильтров: Regular/Irregular, AR/ER/IR первыми.

### Backend
- FastAPI + uvicorn + psycopg2-binary.
- 4 REST-эндпоинта: categories, verbs, persons, forms.
- Health-check endpoint.
- CORS middleware.
- Pydantic-модели для типизации ответов.

### Документация
- README.md с инструкциями по запуску.
- TZ-specification.md — техническое задание.
- DB-SCHEMA.md — схема базы данных.
- CHANGELOG.md — история изменений.
