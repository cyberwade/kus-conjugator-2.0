# Kus Conjugator 2.0 — Техническое задание

## 1. Общая информация

### 1.1. Назначение
Веб-тренажёр спряжения испанских глаголов. Пользователь выбирает грамматические категории, получает список глаголов и тренируется вводить личные формы спряжения.

### 1.2. Целевая аудитория
- Изучающие испанский язык (самостоятельно или с преподавателем).
- Преподаватели, использующие тренажёр как вспомогательный инструмент.

## 2. Технологический стек

| Слой | Технологии |

| **Backend** | Python 3.x, FastAPI, uvicorn, psycopg2-binary, python-dotenv, pydantic |

## 3. Функциональные требования

### 3.1. Блок фильтров

#### Описание
Пять уровней фильтрации с мультивыбором:

| Уровень | Поле БД | Заголовок UI |
|---------|---------|-------------|
| 1 | `mood` | Наклонение |
| 2 | `tense` | Время |
| 3 | `verb_category` | Категория глагола |
| 4 | `verb_type` | Тип глагола |
| 5 | `infinitive` | Глагол |

#### Порядок отображения значений
- Уровни 1–4: алфавитная сортировка (испанская локаль).
- **Категория глагола**: Regular, Irregular, затем остальные.
- **Тип глагола**: AR, ER, IR, затем остальные по алфавиту.
- **Глагол**: алфавитная сортировка сверху вниз (column-fill).

#### Фильтр глаголов (уровень 5 — «Глагол»)
- Секция сворачиваемая (collapsible): заголовок «Глагол» с треугольной стрелкой (▸ / ▾).
- В развёрнутом виде — многоколоночная сетка (`column-count`): 6 колонок на десктопе, 4 на планшете (< 1024px), 3 на мобильных (< 600px).
- Глаголы выстроены в алфавитном порядке сверху вниз по колонкам.
- Счётчик выбранных глаголов (badge) рядом с заголовком, всегда присутствует в DOM (при пустом выборе — `opacity: 0`) для предотвращения скачков раскладки.
- Рядом со счётчиком — кнопка сброса (✕) для очистки выбора глаголов.
- Кросс-фильтрация: глаголы, не имеющие категорий при текущих фильтрах уровней 1–4, отображаются как disabled.

#### Логика выбора
- Значения — уникальные непустые значения из таблицы `categories`, нормализованные (`trim`).
- Если на уровне нет выбора — он не ограничивает выборку.
- Если ни на одном уровне нет выбора — список глаголов не загружается.

#### Кросс-фильтрация (disabled-значения)
- Значение `V` уровня `L` недоступно, если среди категорий, удовлетворяющих выбору на остальных уровнях, нет ни одной строки с `L = V`.
- Недоступные значения: отображаются как disabled, не реагируют на клики, имеют tooltip.
- Уже выбранные значения не становятся disabled автоматически.
- Кросс-фильтрация учитывает фильтр глаголов: выбранные глаголы ограничивают доступные значения уровней 1–4 и наоборот.

#### Состояния
- Загрузка: «Загрузка фильтров…»
- Ошибка: красный текст ошибки.

### 3.2. Блок текущего глагола

#### Загрузка
- На основе фильтров формируется список `category_id`.
- Если список пуст — глаголы не загружаются, индекс сбрасывается.
- Backend возвращает записи из `verb_category` с `infinitive` из `verbs`, сортировка: `infinitive`, `category_id`.

#### Отображение
- Счётчик: «N из M».
- Контекст категории — двухстрочный формат для предотвращения скачков раскладки:
  - Строка 1: `mood — tense` (наклонение и время).
  - Строка 2: `verb_category — verb_type` (категория и тип глагола).
  - Пустые значения пропускаются; если строка 2 пуста, отображается неразрывный пробел (`\u00A0`) для сохранения высоты.
- Инфинитив глагола (крупный фиолетовый шрифт, по центру).

#### Навигация
- «← Предыдущий» / «Следующий →» — переход по списку.
- При смене глагола сбрасывается блок спряжения.

#### Состояния пустоты
- Загрузка: «Загрузка глаголов…»
- Нет глаголов при выбранных фильтрах: «Нет глаголов по выбранным фильтрам».
- Нет выбранных фильтров: «Выберите фильтры для начала тренировки».

### 3.3. Блок спряжения

#### Загрузка лиц
- Лица загружаются один раз при открытии (GET `/api/persons`).
- Для каждого `person_id` — метка `label` (yo, tú, él/ella, ...).

#### Форма ввода
- Для каждого лица: подпись слева, поле ввода (text input, placeholder «Введите спряжение…», цвет placeholder — `#ccc`).
- Одноколоночная раскладка, максимальная ширина 400px, по центру.

#### Кнопка «Проверить»
- Всегда доступна (не требует ввода).
- Запрашивает правильные формы с backend (GET `/api/forms`).
- Сравнение: `trim().toLowerCase()` для пользовательского ответа и эталона.
- После проверки поля становятся read-only.

#### Отображение результатов
- Правильный ответ: зелёная рамка/фон.
- Неправильный ответ: красная рамка/фон + правильная форма над полем ввода.
- Правильные ответы при ошибке отображаются **над полем ввода** (в строке с меткой лица), выровнены на 113px от левого края (примерно 30px после самой длинной метки «ellos,ustedes»), зелёный жирный шрифт, совпадающий по стилю с меткой лица. Без префикса «Правильно:».
- Если правильная форма = NULL (отсутствует в БД):
  - Пустое поле, пробел или прочерк (`-`, `–`) → правильный ответ (зелёная подсветка).
  - Любой другой ввод → неправильный ответ, подсказка «–».
- Сводка: «Результат: X из Y» — по центру, серый мелкий текст, под кнопками. Блок `summary` всегда присутствует в DOM (при отсутствии результатов — `visibility: hidden`) для предотвращения скачков раскладки.

#### Кнопка «Повторить»
- Всегда доступна.
- Сбрасывает ответы, результаты, возвращает поля к редактируемому состоянию.

#### Навигационные стрелки (← →) в блоке спряжения
- Кнопки-стрелки «←» и «→» расположены по бокам от кнопок «Проверить» / «Повторить».
- Круглые (36px, border-radius: 50%), тот же стиль, что у навигационных кнопок.
- Позволяют переключать глагол, не возвращаясь к блоку навигации.

#### Стилизация кнопок
- Обе кнопки («Проверить», «Повторить») — градиентные (фиолетовый), по центру.

#### Ошибки
- Ошибки загрузки лиц или форм отображаются в блоке, без падения приложения.

## 4. API

### 4.1. Общие требования
- REST over HTTP, JSON.
- Backend: FastAPI + uvicorn.
- CORS: разрешены все origins.
- Health-check: `GET /health` → `{"status": "ok"}`.

### 4.2. Эндпоинты

#### GET `/api/categories`
Все категории для построения фильтров.
```json
[
  {
    "category_id": 1,
    "mood": "Indicativo",
    "tense": "Presente",
    "verb_category": "Regular",
    "verb_type": "AR"
  }
]
```

#### GET `/api/verbs?categoryIds=1,2`
Глаголы по выбранным категориям.
```json
[
  {
    "verb_category_id": "5_10",
    "verb_id": 5,
    "category_id": 10,
    "infinitive": "hablar"
  }
]
```

#### GET `/api/persons`
Список лиц.
```json
[
  { "person_id": 1, "label": "yo" },
  { "person_id": 2, "label": "tu" }
]
```

#### GET `/api/forms?verbId=5&categoryId=10`
Правильные формы спряжения.
```json
[
  { "person_id": 1, "form": "hablo" },
  { "person_id": 2, "form": "hablas" }
]
```

#### GET `/api/all-verbs`
Все глаголы (для фильтра по глаголам).
```json
[
  { "verb_id": 1, "infinitive": "hablar" },
  { "verb_id": 2, "infinitive": "comer" }
]
```

#### GET `/api/verb-category-map`
Маппинг глаголов к категориям (для кросс-фильтрации фильтра глаголов).
```json
[
  { "verb_id": 1, "category_id": 10 },
  { "verb_id": 1, "category_id": 11 }
]
```

## 6. Frontend — стиль интерфейса

### Общий стиль
- Светлая тема, градиентный фон страницы (светло-фиолетовый).
- Карточная вёрстка: блоки оформлены как белые карточки со скруглёнными углами и мягкой тенью.
- Акцентный цвет: фиолетовый градиент (`#6366f1` → `#8b5cf6` → `#a855f7`).

### Заголовок
- «Kus Conjugator 2.0», компактный шрифт (28px), без подзаголовка, без разделительной линии.

### Кнопки фильтров
- Компактные pill-кнопки (border-radius: 999px).
- Active: фиолетовый градиент, белый текст.
- Disabled: пониженная насыщенность, без hover-эффектов.

### Кнопки навигации
- Pill-кнопки с текстом и стрелками.
- Hover: фиолетовая рамка + тень.

### Поля ввода
- Шрифт 16px, скруглённые углы (10px), текст по центру.
- Placeholder «Введите спряжение…» — цвет `#ccc` (светлее стандартного).
- Цветовая индикация: зелёная рамка/фон для правильных, красная — для неправильных.

### Адаптивность
- Desktop: все блоки в один столбец, максимальная ширина 1126px.
- Tablet (< 1024px): уменьшенные шрифты и отступы; фильтр глаголов — 4 колонки.
- Mobile (< 600px):
  - Навигационные кнопки: шрифт 11px, меньше padding.
  - Инфинитив глагола: шрифт 26px (вместо 36px).
  - Поля ввода: padding 5px 10px, шрифт 14px.
  - Карточки: padding 12px 16px 14px, gap 12px.
  - Фильтр глаголов — 3 колонки.
  - Стрелки навигации (← →): 32px (вместо 36px).

## 7. Нефункциональные требования

- **Производительность**: загрузка данных — сотни миллисекунд при типовом объёме.
- **Масштабируемость**: добавление глаголов/категорий без изменения кода.
- **Качество кода**: типизация (Pydantic, TypeScript), компонентный подход.
